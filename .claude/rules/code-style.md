# Стиль кода

## Python (бэкенд)
- Async/await для всех обработчиков и сервисов
- Pydantic BaseModel для валидации запросов/ответов (в `schemas.py` каждого модуля)
- pydantic-settings для конфигурации из переменных окружения (`core/config.py`)
- Комментарии на русском языке
- Типизация параметров и возвращаемых значений
- ruff для линтинга и форматирования (line-length 100, double quotes)

## Архитектурные паттерны (бэкенд)
- **Тонкие роутеры** — роутеры содержат только HTTP-логику, бизнес-логика в сервисах
- **DI через Depends** — зависимости инжектятся через FastAPI `Depends()` + `lru_cache` для синглтонов
- **dependencies.py** — единственное место в модуле, где разрешён кросс-модульный импорт
- **Репозитории** — доступ к БД изолирован в `repository.py`, сервисы не работают с ORM напрямую
- **Стратегии** — паттерн Strategy для контекста (`BaseContextStrategy` ABC + 4 реализации)
- **ABC для провайдеров** — `BaseLLMProvider` абстрагирует LLM, модули зависят от интерфейса

## TypeScript / React (фронтенд)
- Strict TypeScript (`strict: true`)
- Только функциональные компоненты с хуками (useState, useEffect, useRef)
- Интерфейсы (не type aliases) для описания структур данных
- API-вызовы изолированы в `src/api/`
- Типы вынесены в `src/types/`

## Общее
- Язык общения и комментариев — русский
- Имена переменных, функций, компонентов — на английском
- Разделение ответственности: модули не импортируют друг друга, компоненты не делают прямых HTTP-запросов
