# Композиция MCP-инструментов — research_server

## Цель

Создать MCP-сервер `research_server.py` с тремя инструментами (`search_files`, `summarize_text`, `save_to_file`), которые LLM автоматически вызывает последовательно как пайплайн: поиск → обработка → сохранение.

## Архитектура

Один новый MCP-сервер. Существующая инфраструктура (AgentRunner, MCPManager, ChatService, фронтенд) не меняется — agent loop уже поддерживает цепочки до 10 итераций.

## Новый файл: `backend/mcp_servers/research_server.py`

### Инструмент 1: `search_files`

Рекурсивный поиск по содержимому файлов.

**Параметры:**
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `directory` | string | да | Путь к директории для поиска |
| `pattern` | string | да | Regex-паттерн для поиска в содержимом |
| `file_glob` | string | нет | Glob-фильтр файлов (по умолчанию `"*"`) |

**Возвращает:**
```json
{
  "matches": [
    {"file": "app/main.py", "line": 42, "text": "# TODO: добавить middleware"}
  ],
  "total": 1,
  "truncated": false
}
```

**Ограничения:**
- Максимум 50 совпадений (truncated=true если больше)
- Пропускает бинарные файлы, `.git/`, `__pycache__/`, `node_modules/`
- Относительные пути в результатах (от directory)

### Инструмент 2: `summarize_text`

Структурирование и форматирование текста.

**Параметры:**
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `text` | string | да | Входной текст для обработки |
| `max_lines` | integer | нет | Максимум строк в результате (по умолчанию 50) |
| `format` | string | нет | Формат: `"plain"` (по умолчанию) или `"markdown"` |

**Возвращает:**
```json
{
  "summary": "обрезанный/отформатированный текст",
  "stats": {
    "original_lines": 150,
    "result_lines": 50,
    "chars": 3200,
    "truncated": true
  }
}
```

**Логика:**
- Обрезает до max_lines
- Если format="markdown" — оборачивает в markdown-структуру с заголовком и статистикой
- Подсчитывает статистику (строки, символы)

### Инструмент 3: `save_to_file`

Сохранение текста в файл.

**Параметры:**
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `path` | string | да | Путь к файлу для записи |
| `content` | string | да | Содержимое для записи |

**Возвращает:**
```json
{
  "saved": true,
  "path": "/app/data/reports/todo-report.txt",
  "size_bytes": 1024
}
```

**Логика:**
- Создаёт промежуточные директории если не существуют
- Перезаписывает файл если существует
- Запись ограничена директорией `/app/data/` (безопасность)

## Конфигурация

Добавить в `backend/data/mcp_servers.json`:
```json
"research": {
  "command": "python",
  "args": ["mcp_servers/research_server.py"],
  "enabled": true
}
```

## Пайплайн — пример работы

Запрос пользователя: *«Найди все TODO в Python-файлах и сохрани отчёт»*

```
Итерация 1: LLM → search_files(directory="/app", pattern="TODO", file_glob="*.py")
             ← {"matches": [...], "total": 12}

Итерация 2: LLM → summarize_text(text=результат_поиска, max_lines=30, format="markdown")
             ← {"summary": "# Отчёт...", "stats": {...}}

Итерация 3: LLM → save_to_file(path="/app/data/reports/todo-report.md", content=сводка)
             ← {"saved": true, "path": "...", "size_bytes": 856}

Итерация 4: LLM → текстовый ответ пользователю: "Нашёл 12 TODO, отчёт сохранён в ..."
```

Оркестрация полностью на стороне AgentRunner — существующий agent loop без изменений.

## Что НЕ меняется

- `AgentRunner` — уже поддерживает цепочки
- `MCPManager` — уже умеет подключать и маршрутизировать
- `ChatService` — уже интегрирован с агентом
- Фронтенд — MCPPanel показывает серверы, стриминг показывает tool_call/tool_result
- `gigachat_adapter.py` — конвертация схем уже работает

## Проверка

1. Сервер запускается и подключается через MCPManager
2. `search_files` корректно находит файлы по regex + glob
3. `summarize_text` форматирует и обрезает текст
4. `save_to_file` сохраняет файл в разрешённую директорию
5. LLM автоматически вызывает цепочку search → summarize → save
6. Данные корректно передаются между инструментами через контекст LLM
